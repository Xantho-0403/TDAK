# Project Rules

## General

Do not break existing functionality.

Do not rewrite the entire project.

Implement features incrementally.

---

## Code Quality

Avoid duplicated code.

Prefer readable code.

Refactor only when necessary.

---

## Architecture

Keep the game engine independent from UI.

Separate rendering from game logic.

Separate input from game logic.

---

## Future Compatibility

Design new systems to support:

- Statistics
- Opener
- Repertoire
- Midgame
- Puzzle

---

## Development Process

Before coding:

1. Analyze the current project.
2. Explain the implementation plan.
3. Wait for approval.

Never skip these steps.

## Build Policy

During development, AI must NEVER edit:

- offline_playground.html

This file is a generated build artifact.

Only modify:

- index.html
- css/*
- js/*

The standalone file must only be regenerated by running:

npm run build

after implementation is complete.
